# TRD：安全、隐私与测试策略

## 安全边界

MVP-A 处理教师上传的课件、可能存在的学生信息、模型凭据和生成内容。默认假设上传文件不可信、模型输出不可信、分享链接可能泄露、Worker 会重复执行。

## 身份与权限

1. 未登录用户只可查看固定公开示例；上传、生成、编辑、重写、导出和模型管理均需登录。
2. 所有项目查询必须同时验证 `owner_user_id`，禁止只凭 UUID 读取。
3. 模型配置要求 `model_admin`；API 从可信身份上下文获取 actor，不接受客户端传入 user ID/role。
4. 对资源不存在和无访问权统一返回 `404 RESOURCE_NOT_FOUND`，降低 ID 枚举风险。
5. 删除项目、模型配置变更、版本恢复和后期分享/发布必须写不可变审计日志。

## 文件安全

| 控制 | 要求 |
| --- | --- |
| 限额 | 单文件 ≤ 50 MiB，≤ 200 页/幻灯片，每个快速会话一个主文件 |
| 类型 | 扩展名、MIME、magic bytes 和文件结构必须一致；只允许 PPTX/PDF |
| 压缩包 | PPTX 解压设置展开大小、文件数量、递归层级和压缩比上限，阻止 zip bomb 与路径穿越 |
| 扫描 | 临时私有区完成恶意文件扫描后才能解析；感染文件隔离并清理 |
| 解析 | 解析器运行在无外网、最小权限、受 CPU/内存/时间限制的隔离进程 |
| 存储 | 私有 Bucket、服务端加密、object key 随机化；数据库不保存长期签名 URL |
| 下载 | 登录鉴权后签发不超过 10 分钟的下载 URL；响应使用安全 Content-Disposition |

原始文件名只用于展示和下载建议名，不能直接拼接本地路径、Bucket key、命令行参数或 HTML。

## 外部模型与敏感数据

1. 文件解析后、调用外部模型前检测姓名、手机号、邮箱、学号、成绩、班级名单等敏感内容。
2. 默认从 Prompt 删除与生成目标无关的个人信息；无法安全脱敏且任务必须依赖该信息时，阻止调用并提示教师处理。
3. 只允许管理员批准的模型供应商、区域和数据策略。每次调用记录 `data_policy`、模型和目的。
4. 优先选择不将 API 数据用于训练且支持最小保留期的服务配置；具体供应商条款由上线环境单独审核。
5. 模型日志只保存输入指纹、结构化统计和脱敏错误，不保存完整课件、完整 Prompt 或学生明文。
6. Prompt injection 不能获得模型凭据、对象存储访问或任意 Tool 权限；模型输出始终经过 Schema 校验和渲染净化。

## 模型凭据

1. API Key 使用 KMS/密钥服务或带密钥版本的 envelope encryption；应用数据库不保存主加密密钥。
2. 仅 AI Gateway 运行身份拥有解密权限，调用时短暂驻留内存，不写磁盘、trace 或错误消息。
3. 前端只返回 `credential_status` 和末四位掩码；更新时使用 write-only 字段。
4. 支持轮换、禁用、删除和最小连通性测试。删除配置后清除密文并保留不含秘密的审计记录。

## HTML、导出与内容安全

1. 模型不得输出可直接执行的 HTML/JavaScript；只输出白名单 Schema，模板负责渲染。
2. 富文本渲染禁止脚本、事件属性、iframe、任意外链资源和危险 URL scheme。
3. PDF/DOCX/Markdown 导出使用固定模板和转义规则；公式、链接、图片分别做协议和来源校验。
4. 教育事实不能仅凭“Schema 通过”视为正确；MVP-A 显示来源和复核提示，教师确认后才能标记为已审核。

## MVP-B 分享安全

MVP-A 不提供学生分享。MVP-B 必须同时满足：

1. 至少 256 bit CSPRNG token，只返回一次原始值，数据库保存 HMAC/SHA-256 结果。
2. 默认有效期 7 天，最长 30 天；支持立即撤回并记录访问审计。
3. 分享清单固定到明确的 output/content block 版本，未确认或 blocking finding 未解决时禁止发布。
4. 页面设置合理的缓存、搜索索引和引用策略，不采集非必要学生数据。
5. 限速并监控 token 枚举、异常访问量和批量下载。

## 数据保留与删除默认值

| 数据 | 默认保留 | 删除行为 |
| --- | --- | --- |
| 未生成的快速会话和临时文件 | 24 小时 | 自动物理删除临时对象，保留最小清理日志 |
| 已生成会话聚合记录 | 30 天 | 删除 source text 密文和临时错误详情 |
| 项目、内容版本和导出 | 用户删除前 | 删除后立即不可访问，7 天内清理在线对象 |
| 已过期导出对象 | 30 天 | 自动删除对象，保留导出元数据和 hash |
| 模型调用明细 | 90 天 | 删除逐次明细；可保留不可回溯到个人的聚合成本 |
| 安全和高影响审计日志 | 180 天 | 到期按环境策略删除或归档 |
| 备份 | 30 天滚动 | 到期覆盖；恢复后重新执行删除 tombstone |

这些是产品默认值。机构部署可以缩短，延长前必须经过隐私和合规评审。用户删除项目后返回删除请求 ID，并允许查询清理状态。

## 日志与可观测性

1. 日志默认禁止文件正文、source text、完整 Prompt、Authorization、Cookie、API Key、签名 URL 和分享 token。
2. request/task/model call 使用同一 trace 关联；面向用户的 `request_id` 不暴露内部拓扑。
3. 指标标签不得使用 user ID、project ID、文件名或高基数字段。
4. 记录队列积压、任务失败率、重试率、模型错误率、导出失败率、扫描失败和异常鉴权事件并设置告警。

## 测试策略

| 类型 | 必测内容 |
| --- | --- |
| 单元测试 | CourseIR、ContentBlock、状态流转、Schema、权限策略、错误映射 |
| 契约测试 | OpenAPI request/response、错误码、枚举和生成客户端兼容性 |
| 集成测试 | 文件/文本入口、解析、生成、编辑、重写、审核、导出、删除 |
| 幂等与并发 | 重复生成、重复重试、Worker 重放、版本冲突、同块并发重写 |
| 生成质量 | 准确性、来源支持、难度、题目结构和教师可用性 |
| 权限安全 | 越权读取、对象存储 URL、管理员接口、日志泄密、恶意文件、HTML 注入 |
| 性能可靠性 | 基准输入 P95、队列积压、依赖超时、Worker 崩溃恢复、部分成功 |
| 数据生命周期 | 会话过期、项目删除、导出过期、备份恢复后的删除重放 |

测试数据集、阈值和发布门禁见 [MVP-A 测试与验收规范](../testing/mvp-a-acceptance.md)。

## Definition of Done

1. 正常、异常、权限、幂等和并发路径均有自动化测试。
2. 实现通过 OpenAPI 校验，客户端由同一契约生成。
3. 长任务有状态、有限重试、死信、失败原因和人工恢复入口。
4. 模型调用、对象存储和应用日志经过泄密测试。
5. 用户可见文案符合语言映射表。
6. 基准质量与性能门禁通过，且不存在未关闭的高危安全问题。
