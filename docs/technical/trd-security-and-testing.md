# TRD：安全合规与测试策略

## 安全要求

| 领域 | 要求 |
| --- | --- |
| 文件安全 | 上传前校验类型、大小、空文件和恶意文件风险 |
| 权限 | 教师只能访问自己的草稿和已授权资源 |
| 学生数据 | 分享页默认免登录但使用不可猜测 token，可撤回、可过期 |
| 模型 Key | 普通教师不可见，仅管理员配置；日志必须脱敏 |
| 版权 | 上传区展示隐私和版权提示；导出/分享前提示教师确认 |
| 审计 | 发布、覆盖版本、推送学生、模型配置变更必须记录审计日志 |

## 测试策略

| 类型 | 覆盖内容 |
| --- | --- |
| 单元测试 | CourseIR、ContentBlock、字段校验、状态映射 |
| 集成测试 | 上传、解析、生成、重写、导出、分享 |
| 契约测试 | API request/response、Schema、错误码 |
| 生成质量测试 | 题目答案一致性、来源页码、难度匹配 |
| 权限测试 | 私有草稿、分享链接、管理员模型配置 |
| 性能测试 | 上传到首版可预览 ≤ 3 分钟 |
| 可观测测试 | 任务状态、日志、指标、埋点是否完整 |

## Definition of Done

1. 功能路径、异常路径和权限路径均有测试。
2. 关键 API 有契约示例。
3. 长任务有状态、重试和失败原因。
4. 模型调用有日志脱敏和成本统计。
5. 用户可见文案符合语言映射表。
