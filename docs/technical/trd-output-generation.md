# TRD：内容产物生成与渲染

## 目标

把 CourseIR 和 ContentBlock 转换成教师可用、学生可学、系统可追溯的教学产物。MVP-A 支持学习要点、随堂小测、教师讲解提示、教案初稿，单次默认生成项由用途决定；HTML 仅承载登录后的在线预览。学生分享和基础 PPTX 属于 MVP-B。

## 产物优先级

| 产物 | MVP-A | MVP-B | 标准版 | 说明 |
| --- | --- | --- | --- | --- |
| 学习要点 | 必做 | 增强学生自测 | 增强模板 | 面向学生复习。 |
| 随堂小测与解析 | 必做 | 增强错因解释 | 增强题型和批量 | 面向课堂检测。 |
| 教师讲解提示 | 必做 | 增强课堂节奏 | 增强风格模板 | 面向上课话术。 |
| 教案初稿 | 必做，含基础版本恢复 | 增强审核发布 | 增强可视化版本对比与批量恢复 | 面向备课沉淀。 |
| HTML 页面 | 登录后在线预览 | 学生免登录分享页 | 可扩展主题 | 不作为独立生成产物。 |
| PPTX | 不承诺高质量 | PPT 大纲或基础 PPTX | 高质量 PPTX | 避免吞噬 MVP 资源。 |

## 测试卡字段

| 字段 | 说明 |
| --- | --- |
| `question` | 题干 |
| `type` | 题型，如单选、多选、判断、填空、简答、案例分析 |
| `options` | 选项 |
| `answer` | 正确答案 |
| `explanation` | 解析 |
| `difficulty` | 难度 |
| `knowledge_point_id` | 关联知识点 |
| `source_refs` | 来源页码 |

## 学习要点结构

1. 本课概览。
2. 核心知识点。
3. 重点公式或概念。
4. 易错点。
5. 记忆技巧。
6. 课后自测。

## 教案结构

1. 教学目标。
2. 教学重点。
3. 教学难点。
4. 教学准备。
5. 教学过程。
6. 课堂互动。
7. 板书设计。
8. 作业设计。
9. 教学反思。

## HTML 渲染要求

1. 模型输出页面结构 JSON，模板负责渲染。
2. 支持响应式布局。
3. 不允许直接信任模型输出任意脚本。
4. MVP-A 仅作为登录后的在线预览；MVP-B 才允许生成学生分享页。

## PPTX 渲染要求

1. 使用 Slide JSON + PPT 渲染器。
2. 支持教师授课版、学生简版、复习版。
3. 模板、主题、字体兼容和图文布局作为标准版重点能力。

## 导出要求

1. PDF/DOCX/Markdown 是 MVP-A 优先导出格式。
2. 导出文件必须记录内容版本、模板版本和生成时间。
3. 所有导出均应可追溯到 CourseIR 版本和来源页码。
4. MVP-A 的 HTML 在线预览、PDF、DOCX 和 Markdown 共用 `packages/renderer-html` 内的受控 Document AST 与模板版本；格式 adapter 不得散落在 Worker。`packages/renderer-ppt` 只承载 MVP-B/标准版 PPTX。
