# TRD：AI 编排与渲染导出

## AI 生成链路

```text
文件解析
  → 课程信息识别
  → CourseIR 构建
  → 生成学习要点 / 小测 / 教师提示 / 教案初稿
  → Schema 校验
  → 自动修复或降级
  → 内容块版本保存
  → 质量审校
  → 渲染导出
```

## Prompt 管理要求

每个生成能力必须记录：

- `prompt_key`
- `prompt_version`
- 输入变量
- 输出 Schema
- 校验和自动修复策略
- 失败降级策略
- 关联内容块版本

## 模型网关要求

所有模型调用必须经过统一网关：

1. 屏蔽不同供应商差异。
2. 支持管理员配置模型池和 Key。
3. 记录 token、成本、延迟、失败原因。
4. 对日志中的学生数据、文件内容、Key 进行脱敏。
5. 支持重试、fallback 和限流。

## 渲染导出原则

1. 模型只生成结构化内容，不直接生成最终文件。
2. HTML 在 MVP-A 用作登录后的在线预览页；学生分享页从 MVP-B 开始。
3. PDF/DOCX/Markdown 是 MVP-A 优先导出格式。
4. PPTX 放入 MVP-B 或标准版，根据模板成熟度逐步开放。
5. 导出产物必须记录内容版本、模板版本和生成时间。
