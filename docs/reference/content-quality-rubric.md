# 内容质量评分标准

## 教学质量

| 维度 | 说明 | MVP 判断方式 |
| --- | --- | --- |
| 准确性 | 知识点、公式、答案是否正确 | 教师审核 + 题目答案一致性检查 |
| 来源可追溯 | 内容是否能回到原课件页码 | 每个关键内容块保留来源页码 |
| 难度匹配 | 是否适合识别出的年级和学生水平 | 低置信字段提示教师确认 |
| 可讲授性 | 教师是否能直接拿来讲 | 是否导出、是否二次使用 |
| 可学习性 | 学生是否能理解和自测 | 学习要点阅读率、自测完成率 |

## 产品质量

| 指标 | MVP 目标 |
| --- | --- |
| 首次可预览时间 | ≤ 3 分钟 |
| 首次导出率 | ≥ 40% |
| 无需大改即可使用比例 | ≥ 60% |
| 局部重写使用率 | ≥ 30% |
| 首次无需 API Key | 100% |

## 工程质量

| 维度 | 要求 |
| --- | --- |
| 结构化输出 | 所有模型输出必须经过 Schema 校验或等价完整性检查 |
| 可追溯 | 保留文件、页码、Prompt 版本、模型、成本、延迟和失败原因 |
| 可恢复 | 长任务可重试、可查询、可展示失败原因 |
| 可观测 | 核心链路必须有日志、指标和事件埋点 |
