# 验收清单与风险登记

## MVP-A 验收清单

| 类别 | 验收标准 |
| --- | --- |
| 首次体验 | 登录教师不手动创建项目、不配置模型即可上传 PPTX/PDF 或提交文本需求 |
| 生成速度 | 基准输入下，从点击生成到首个默认产物可预览的 P95 ≤ 3 分钟 |
| 生成产物 | 系统支持四类产物；每次按用途矩阵生成默认产物，且至少成功一个主要产物 |
| 编辑体验 | 可对任一内容块局部编辑、重写、删除、确认和按明确版本恢复；每次动作产生新版本 |
| 审核 | 展示来源页码、低置信字段和题目答案复核提示 |
| 导出 | 支持 PDF/DOCX/Markdown 导出 |
| 埋点 | MVP-A 覆盖访问、示例、登录、来源提交、生成、预览、重写、导出和第二次复用；分享从 MVP-B 开始 |
| 安全 | 普通教师不可见 API Key、Base URL、Token |

## 主要风险

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| MVP 范围膨胀 | 延迟真实教师验证 | 严格执行 MVP-A 边界，高质量 PPTX、广场、复杂 Agent 后置 |
| 术语外露 | 教师上手困难 | 所有教师端文案遵守用户语言映射表 |
| 生成质量不稳定 | 教师不敢导出或分享 | 保留来源页码、审核面板、局部重写和人工确认 |
| PPTX 排版成本过高 | 吞噬 MVP 资源 | MVP-A 仅提供导出文档；PPTX 放入 MVP-B/标准版 |
| 模型成本不可控 | 商业化亏损 | 模型网关统一统计 token、成本、延迟，绑定套餐额度 |
| 分享页权限风险 | 学生资料泄露 | 使用不可猜测 token、有效期、撤回、审计日志 |

## 发布前硬门槛

1. 首次体验无需 API Key 为 100%。
2. 上传失败、解析失败、生成失败都有教师可理解的处理建议。
3. 所有高影响动作都有确认：发布、覆盖版本、推送学生。
4. 导出内容能追溯到内容版本和来源页码。
5. 文本需求和文件上传两种入口均有端到端自动化用例。
6. 详细输入基线、百分位、质量阈值和失败分级以 [MVP-A 测试与验收规范](../testing/mvp-a-acceptance.md) 为准。
7. `partial_success`/`failed` 重试会创建新 generation run，旧任务和成功版本保持可追溯。
