# 执行路线图

## 阶段 0：文档与范围收敛

目标：完成 PRD / TRD / 执行文档拆分，统一 MVP-A 范围和教师端语言。

交付：

- 文档入口 README。
- PRD 主链路。
- TRD 架构与接口草案。
- 任务卡模板。
- 术语映射表。

## 阶段 1：MVP-A 教师备课闭环

目标：验证老师是否愿意上传课件并导出生成结果。

交付：

1. 快速生成入口。
2. PPTX/PDF 上传与解析。
3. 课程信息自动识别。
4. 学习要点、随堂小测、教师讲解提示、教案初稿。
5. 局部编辑和局部重写。
6. 来源页码和审核提示。
7. PDF/DOCX/Markdown 导出。
8. 基础埋点和增长漏斗。

## 阶段 2：MVP-B 上课包闭环

目标：验证第二天课堂可用。

交付：

1. PPT 大纲或基础 PPTX。
2. 学生免登录分享页。
3. 3-5 题自测、答案解析、错因解释。
4. 分享撤回与发布确认。
5. 教师审核面板增强。

## 阶段 3：标准版

目标：服务高频教师和教研个人付费用户。

交付：

1. 高质量 PPTX。
2. 模板包和主题。
3. 批量导出。
4. 历史版本对比和恢复。
5. 高级质量审校。

## 阶段 4：机构版与生态

目标：服务学校/教培机构，并逐步建设资源生态。

交付：

1. 组织、角色、权限。
2. 机构模型池、额度、审计。
3. 校本资源库和模板。
4. 私有化部署。
5. 机构内协作。
6. 后期再评估公开广场、复制为我的版本、关注更新和 Chat 助教。
