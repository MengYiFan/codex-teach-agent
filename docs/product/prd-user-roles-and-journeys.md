# PRD：用户角色与核心旅程

## 用户角色

| 角色 | 核心诉求 | 首次进入阶段与权限 |
| --- | --- | --- |
| 教师 | 快速生成上课材料、复习要点和练习题 | MVP-A：上传/文本生成、编辑、审核、导出；MVP-B：分享 |
| 教研员 | 按统一模板生成和审核资源 | 标准版/机构版：管理模板、审核资源、共享课程 |
| 机构管理员 | 控制模型、费用、权限和数据 | MVP-A 仅系统模型管理员；完整组织/额度/审计进入机构版 |
| 学生 | 查看教师发布的学习要点和测试卡 | MVP-B：免登录访问分享页、完成自测 |

## MVP-A 教师主旅程

```text
教师准备明天课程
  → 上传已有 PPT/PDF，或直接输入不少于 10 个字符的备课需求
  → 选择“明天上课”或“发学生复习”
  → 系统按用途生成 2-3 类默认材料，教师可追加其他 MVP-A 产物
  → 教师确认低置信字段
  → 局部重写题目或讲解提示
  → 导出 PDF/DOCX/Markdown
```

该旅程要求教师已登录；未登录用户可以查看示例，但不能上传真实资料。

## MVP-B 学生分享旅程

```text
教师生成学生分享链接
  → 学生免登录打开
  → 查看学习要点
  → 完成 3-5 道自测
  → 查看答案解析和错因解释
  → 回看相关知识点
```

## 后期旅程

公开广场、复制为我的版本、关注更新、围绕资料提问等能力放入后期生态，不进入 MVP-A 主线。
