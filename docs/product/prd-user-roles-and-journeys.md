# PRD：用户角色与核心旅程

## 用户角色

| 角色 | 核心诉求 | MVP 权限 |
| --- | --- | --- |
| 教师 | 快速生成上课材料、复习要点和练习题 | 上传课件、生成内容、编辑、导出、分享 |
| 教研员 | 按统一模板生成和审核资源 | 管理模板、审核资源、共享课程 |
| 机构管理员 | 控制模型、费用、权限和数据 | 管理用户、模型 Key、额度、审计 |
| 学生 | 查看教师发布的学习要点和测试卡 | 免登录访问分享页、完成自测 |

## MVP-A 教师主旅程

```text
教师准备明天课程
  → 上传已有 PPT/PDF
  → 选择“明天上课”或“发学生复习”
  → 系统生成四类材料
  → 教师确认低置信字段
  → 局部重写题目或讲解提示
  → 导出 PDF/DOCX/Markdown
```

## MVP-B 学生分享旅程

```text
教师生成学生分享链接
  → 学生免登录打开
  → 查看学习要点
  → 完成 3-5 道自测
  → 查看答案解析和错因解释
  → 回看相关知识点
```

## 后期旅程

公开广场、复制为我的版本、关注更新、围绕资料提问等能力放入后期生态，不进入 MVP-A 主线。
