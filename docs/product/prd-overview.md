# PRD 总览：10 分钟备课助手

## 产品定位

首版不应被包装成复杂的“教学内容生成平台”，而应定位为面向一线教师的 **10 分钟备课助手**：

```text
上传课件或描述需求，3 分钟内看到首个可用结果：
✓ 学生复习要点
✓ 随堂小测与答案解析
✓ 教师讲解提示
✓ 可编辑教案初稿
```

## 核心用户心智

教师端首要心智是：

```text
我有一份课件
  → 帮我快速变成明天能用的教学材料
  → 我能改其中某一段或某几道题
  → MVP-A 我能导出；MVP-B 我还能分享给学生
```

教师端不应让普通教师先理解项目、CourseIR、Schema、异步任务、Base URL、API Key、Token、fork、watch 或 Agent Workflow。

## 产品原则

1. **上传即生成**：上传 PPTX/PDF 或输入一句备课需求后，系统自动创建后台草稿。
2. **先给初稿，后补信息**：先让教师看到可编辑结果，再提示确认学科、年级、课时、学生水平等字段。
3. **用途驱动，而非功能驱动**：首屏使用“明天上课 / 发学生复习 / 出随堂小测 / 写教案”。
4. **默认隐藏模型配置**：教师不配置模型；系统或机构管理员预配置模型、Key、额度和成本策略。
5. **局部重写优先**：不追求一次生成完美，必须支持“改简单点 / 换例子 / 换题 / 重新生成这一段”。
6. **教师审核兜底**：发布、覆盖版本、推送学生等高影响动作必须有人工确认。

## 首版成功标准

| 指标 | MVP 目标 |
| --- | --- |
| 基准输入首次有意义预览时间 | 热运行 P95 ≤ 3 分钟 |
| 首次成功生成用户 7 天内导出率 | ≥ 40% |
| 按质量标准判定无需大改比例 | ≥ 60% |
| 首次成功生成用户 7 天内局部重写比例 | ≥ 30% |
| 首次体验无需配置 API Key | 100% |

## 不做什么

MVP-A 不做高质量 PPTX、完整教学 HTML 产品、教案广场、学生账号、复杂 Agent 生命周期、普通教师模型配置、机构知识库、移动端 App。

指标定义、样本量和测试环境见 [MVP-A 测试与验收规范](../testing/mvp-a-acceptance.md)。
