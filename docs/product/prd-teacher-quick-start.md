# PRD：教师快速生成入口

## 目标

让已登录教师无需手动创建项目、无需配置模型、无需阅读教程，即可上传课件或输入需求，并获得可编辑、可导出的教学材料初稿。学生分享属于 MVP-B。

## 首次使用流程

```text
上传课件或输入需求
  → 选择用途：明天上课 / 发学生复习 / 出随堂小测 / 写教案
  → 系统解析并自动识别课程信息
  → 直接生成低风险初稿
  → 在结果顶部展示“我猜你是：学科 / 年级 / 课时 / 学生水平，可修改”
  → 教师修改后局部重写或导出
```

## 首屏组成

| 区域 | 默认展示 | 交互 | 异常/空状态 |
| --- | --- | --- | --- |
| 顶部价值说明 | “上传课件或描述需求，生成复习要点、随堂小测和讲解提示” | 点击“看示例”进入示例项目 | 未登录用户可看示例；点击生成时要求登录 |
| 上传区 | 支持拖拽 PPTX/PDF，显示大小限制和隐私提示 | 选择文件后立即校验类型、大小、是否空文件 | 不支持格式时提示“请另存为 PDF/PPTX 后上传” |
| 文本需求 | 示例：“准备高一物理牛顿第二定律的 45 分钟随堂课” | 与文件二选一，也可作为文件补充说明 | 少于 10 个字符时提示补充学科、主题或用途 |
| 用途卡片 | 明天上课、发学生复习、出随堂小测、写教案 | 单选，默认选中“明天上课”；用途决定默认产物 | 未选择时按钮不可提交并解释用途影响输出 |
| 高级设置 | 默认收起 | 可填写年级、课时、学生水平、模板偏好 | 可全部留空，系统使用可解释默认值 |
| 生成进度 | 解析课件、识别课程、生成初稿、质量检查 | 分阶段展示已完成产物，允许关闭页面稍后查看 | 失败时定位到文件页码或生成步骤 |
| 结果确认卡 | “识别到：学科/年级/课时/学生水平” | 修改后可选择“仅更新后续生成”或“重新生成全部” | 低置信字段显示“请确认”，不阻塞预览 |

## 字段策略

| 字段 | 教师端策略 | 说明 |
| --- | --- | --- |
| 输出用途 | 首次唯一建议必选 | 用场景卡片替代表单字段 |
| 备课需求 | 文本模式必填，文件模式选填 | 10-2000 字；不得只包含空白字符 |
| 课程名称 | 自动识别，允许修改 | 低置信度时再追问 |
| 学科 | 自动识别，允许修改 | 不作为上传前置阻断 |
| 年级 | 自动识别，无法识别时留空 | 年级与难度是不同字段，不使用“普通难度”代替年级 |
| 课时 | 默认 45 分钟 | 可在结果页调整并局部重写 |
| 学生水平 | 默认普通 | 可用“基础 / 普通 / 提高”快速切换 |

## 教师端状态文案

`uploading` 是浏览器本地上传状态；服务端从 `source_validating` 开始记录会话状态。`generating_notes`、`generating_quiz` 等是 `generating` 下的 `current_step`，不是独立会话状态。

| 会话状态 | 教师端文案 | 可操作动作 |
| --- | --- | --- |
| `source_validating` | 正在检查课件 | 取消上传 |
| `source_ready` | 资料已准备好 | 开始生成 / 修改用途 |
| `queued` | 已加入生成队列 | 稍后查看 / 取消 |
| `parsing` | 正在识别课件内容，预计 1 分钟 | 稍后查看 |
| `detecting_context` | 正在识别课程信息 | 跳过并先生成 |
| `generating` | 正在生成教学材料；按当前步骤显示要点/小测/教案等进度 | 查看已完成内容 |
| `reviewing_quality` | 正在检查题目答案和来源页码 | 查看草稿 |
| `partial_success` | 部分内容已生成 | 查看或导出成功产物 / 重试失败产物 |
| `succeeded` | 教学材料已生成 | 查看 / 编辑 / 重写 / 恢复 / 导出 |
| `failed` | 生成失败：说明原因和建议处理方式 | 重试 / 更换文件 / 联系支持 |
| `cancelled` | 已取消生成 | 重新开始 |

## 验收标准

1. 未登录用户可查看示例；发起真实生成前必须登录，登录后无需手动创建项目。
2. 文件上传成功或文本需求校验通过后，会话进入 `source_ready`；点击生成时原子创建后台草稿和首个任务，重复提交不会创建重复项目。
3. 教师首次体验不需要配置 API Key、Base URL 或模型名称。
4. 课程信息识别置信度低时，以一句自然语言追问或结果页提示处理，不用完整表单阻断。
5. 生成过程使用教师可理解文案，例如“正在解析课件”“已生成学习要点”“测试题还在生成中”。
6. 文本模式无需上传文件即可完成生成；文件模式允许附加文本说明。
7. MVP-A 页面不出现学生分享、发布或撤回入口。
8. 刷新或重新打开尚未生成的会话后，仍可看到已上传文件、用途、上下文和可继续执行的动作。

完整状态与异常流转见 [快速生成状态机 TRD](../technical/trd-quick-start-state-machine.md)。
