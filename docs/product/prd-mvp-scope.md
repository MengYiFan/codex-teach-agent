# PRD：MVP 范围与版本边界

## MVP-A：教师备课最小闭环

目标：验证老师是否愿意上传真实课件并使用生成结果。

| 范围 | 内容 |
| --- | --- |
| 用户 | 单个一线教师 |
| 入口 | 登录教师上传 PPTX/PDF，或输入一句不少于 10 个字符的备课需求；未登录用户只能查看示例 |
| 必选项 | 选择用途：明天上课 / 发学生复习 / 出随堂小测 / 写教案 |
| 自动识别 | 课程名、学科、年级、课时、学生水平 |
| 首版产物 | 产品支持学习要点、随堂小测与答案解析、教师讲解提示、教案初稿；单次默认产物由用途决定 |
| 编辑能力 | 内容块编辑、局部重写、快捷指令 |
| 审核能力 | 来源页码、低置信提示、题型结构/答案格式检查和教师确认；不承诺自动事实审校 |
| 导出 | PDF、DOCX、Markdown |
| 明确不做 | 高质量 PPTX、完整 HTML 产品、广场、学生账号、复杂 Chat、普通教师模型配置 |

### 用途与默认产物

| 用途 | 默认生成 | 可追加 |
| --- | --- | --- |
| 明天上课 | 教师讲解提示、教案初稿、学习要点 | 随堂小测 |
| 发学生复习 | 学习要点、随堂小测与解析 | 教师讲解提示 |
| 出随堂小测 | 随堂小测与解析 | 学习要点 |
| 写教案 | 教案初稿、教师讲解提示 | 学习要点、随堂小测 |

四类能力都属于 MVP-A，但不要求每次生成全部四类。这样用途选择会真正影响时延和成本。`preferred_outputs` 可覆盖默认值，但至少包含一个输出。

## MVP-B：第二天课堂可用闭环

目标：验证老师能否直接把生成结果用于第二天课堂。

| 范围 | 内容 |
| --- | --- |
| 新增产物 | PPT 大纲或基础 PPTX |
| 学生侧 | 免登录分享链接 |
| 自测 | 3-5 题自测、答案解析、错因解释 |
| 教师侧 | 审核面板、发布确认、分享撤回 |
| 仍不做 | 完整社区、机构知识库、多教案 RAG、移动端 App |

## 标准版

面向高频教师和教研个人付费用户，增强：

1. 高质量 PPTX。
2. 更多模板主题。
3. 无水印与批量导出。
4. 高级编辑器与批量局部重写。
5. 历史版本对比和恢复。
6. 题目答案一致性检查与质量审校。

这里的“质量审校”指跨内容事实一致性、学科正确性和高级难度评估；MVP-A 仍保留基础结构校验和人工复核提示。

## 机构版

面向学校和教培机构，增强：

1. 多租户、组织、角色和权限管理。
2. 机构级模型池、Key 管理和额度限制。
3. 教材、考纲、题库、校本资源知识库。
4. 成本统计、审计日志和数据合规。
5. 班级、课程体系、教师协作和机构内资源共享。
6. 私有化部署和本地模型接入。

## 后期生态能力

教案广场、公开资源 fork/watch、围绕教案的 Chat 助教应作为增长/生态能力，不应并入 MVP 或机构版核心交付。
